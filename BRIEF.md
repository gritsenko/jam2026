## 0. Контекст

Мобильная игра в жанре **Tower Defense / Merge** с механикой **пространственной направленной
синергии** на сетке 3×3. Рабочее название — «Synergy Grid TD».

Полное описание механик — в `docs/concept.pdf`. **Прочитай его для понимания контекста, но на
этом этапе НЕ реализуй боевую логику** (синергия, мердж, нагрузка, волны и т.д.). Задача —
собрать **играбельный каркас**: навигируемая оболочка из трёх сцен, со всем UI боевого экрана
на месте, на замоканных данных, с чистой архитектурой, готовой под последующий геймплей.

Стиль игры: рисованный мультяшный mid-core арт (см. `docs/style_ref.png`) — тёплая
пустыня-каньон, толстые чистые контуры, сочные энергетические свечения (красный/синий/
зелёный/золотой), стимпанк-платформа с рунными слотами, цветовое кодирование карт по
стихиям. UI должен передавать этот вайб, а не выглядеть как дефолтный прототип.

## 1. Технологический стек

- **Vite** + **TypeScript** (strict-режим включён).
- **PixiJS v8** — возьми последнюю стабильную минорную версию (проверь актуальную в npm и
  запинь её в `package.json`). Обоснование выбора: WebGL2/WebGPU-рендер, лучшая
  производительность на мобильных, актуальная архитектура.
- Менеджер пакетов — `npm` (или `pnpm`, если предпочитаешь; зафиксируй один).
- Для анимаций можно использовать встроенные тикеры Pixi или лёгкую твин-библиотеку
  (на твой выбор, обоснуй в README).
- Никаких тяжёлых движков/фреймворков сверху. Чистый Pixi + TS.

## 2. Адаптивность (критично — отдельный приоритет)

Игра портретно-ориентированная, но обязана корректно работать на **телефоне (узкий, ~9:19.5),
планшете (~3:4) и ПК (широкий экран)**.

Реализуй переиспользуемую систему лейаута:

- **Design resolution**: базовое портретное разрешение (напр. 1080×1920), относительно которого
  верстается весь UI.
- **Режим вписывания**: контент масштабируется под экран по принципу contain (без обрезания
  игровой зоны). На широких экранах (ПК/планшет в ландшафте) портретная игровая область
  центрируется как «рамка», а свободные поля заполняются тематическим фоном/декором, а не
  чёрными полосами.
- **Якоря HUD**: верхний бар липнет к верху, рука с картами — к низу, зона Реактора — к правому
  краю. Центральная платформа сохраняет пропорции. То есть HUD реагирует на пропорции экрана,
  а не просто тупо масштабируется целиком.
- **Safe area**: учитывай `env(safe-area-inset-*)` (вырезы/чёлки, скруглённые углы).
- Корректная обработка `resize` и смены ориентации без артефактов и без перезагрузки сцены.
- Резкость на HiDPI: учитывай `devicePixelRatio`.

Вынеси это в модуль `core/ResponsiveLayout` (или аналог). Можешь рассмотреть `@pixi/layout`,
но если берёшь — обоснуй и не делай его обязательной зависимостью всей архитектуры.

## 3. Структура проекта

Предложи и создай чистую структуру, ориентировочно:

```
/
  index.html
  vite.config.ts
  tsconfig.json
  package.json
  BRIEF.md
  docs/                 # concept.pdf, style_ref.png
  src/
    main.ts
    core/               # Game, SceneManager, ResponsiveLayout, AssetLoader
    scenes/             # MainMenuScene, WorldMapScene, BattleScene
    ui/                 # переиспользуемые UI-компоненты (Button, Card, ResourceBar, EnergyGauge...)
    config/             # игровые данные/константы (моки карт, врагов, уровней) — данные ОТДЕЛЬНО от логики
    assets/             # сгенерированные спрайты + манифест
  tools/
    gen_sprite.py       # генератор спрайтов через Nano Banana (см. раздел 6)
    assets.manifest.json
```

---

## 4. Архитектура

- **SceneManager** с переходами между сценами (с базовым fade-переходом).
- **AssetLoader** на базе `Assets` из Pixi + манифест ассетов. Архитектура должна позволять легко
  **подменять плейсхолдер на сгенерированный спрайт** через ключ ассета (placeholder-fallback,
  пока спрайт не сгенерирован).
- Игровые данные (карты, враги, уровни, стоимости) — в `config/` как типизированные структуры,
  отдельно от логики отрисовки.
- Переиспользуемые UI-компоненты в `ui/`.

## 5. Сцены (три)

### 5.1. Главное меню (`MainMenuScene`)
- Тематический фон, логотип/название игры, кнопка **«Начать»** (стилизованная).
- Кнопка ведёт на карту уровней.

### 5.2. Глобальная карта уровней (`WorldMapScene`)
- Карта с узлами-уровнями (5–8 моков), путь между ними, отметка прогресса.
- Тап по доступному узлу запускает боевую сцену. Кнопка «Назад» в меню.

### 5.3. Боевая сцена (`BattleScene`) — главный экран
Свёрстай по `style_ref.png` и разделу 5 концепта. **Обязательные элементы UI (на замоканных данных):**

- **Верхний бар**: счётчик волны (напр. «WAVE 8/20»), индикатор **золота**, индикатор
  **кристаллов**, индикатор **Synergy Points (SP)**, **аватарка героя** в ажурной рамке (справа).
- **Платформа 3×3**: стимпанк-плата с 9 слотами под карты (часть пустые, часть с моковыми
  картами). Слоты кликабельны (пока — заглушка-лог).
- **Индикатор нагрузки энергии**: горизонтальная сегментированная шкала в стиле Red Alert
  (зелёный → жёлтый → красный, + золотой режим Overdrive). Сделай компонент `EnergyGauge`,
  принимающий текущую нагрузку/ёмкость и режим.
- **Рука из 3 карт** внизу: карточные компоненты со стоимостью, иконкой, рамкой по цвету стихии.
- **Зона сброса/сжигания (Реактор)**: у правого края. Реализуй базовый **drag&drop** карты из
  руки: при начале перетаскивания подсвечиваются валидные drop-зоны (слоты сетки + зона
  Реактора), отпускание над зоной — пока заглушка с визуальным фидбеком и логом.
- Зона дорожки врагов вокруг платформы — пока плейсхолдер (можно пара статичных спрайтов
  врагов для атмосферы).

Боевую логику (атаки, синергия, волны, экономика) **не реализуем** — только UI-оболочка,
ввод и моки.

## 6. Спрайт-пайплайн (Nano Banana) — НЕ эмодзи и не CSS-фигуры

Весь игровой арт и UI-иконки — это сгенерированные PNG, **никаких эмодзи, юникод-символов или
сверстанных из примитивов «иконок»**. Источник графики — **Nano Banana** (нативная генерация
изображений Gemini), модель `gemini-3.1-flash-image` (Nano Banana 2; если недоступна — фолбэк
`gemini-2.5-flash-image`). Нужен ключ `GEMINI_API_KEY` из Google AI Studio.

**Создай `tools/gen_sprite.py`** — CLI-генератор со следующими свойствами:
- Аргументы: `--prompt`, `--out`, `--ref` (путь к референсу стиля), опционально `--size`.
- Передаёт `docs/style_ref.png` (или указанный `--ref`) как **референс-изображение** вместе с
  текстовым промптом, с инструкцией «in the exact same art style as the reference» — Nano Banana 2
  хорошо держит стилевую консистентность по референсу.
- Стилевая преамбула в каждом промпте: game asset, single object centered, solid flat #00FF00
  chroma-key background, no drop shadow, clean thick outlines, consistent with reference style.
- **Вырезает хромакей-фон** (зелёный) в прозрачность через PIL после генерации (для пиксель-чётких
  краёв понижай допуск). Сохраняет RGBA PNG.
- Зависимости: `pip install google-genai pillow`.

**Создай `tools/assets.manifest.json`** — список всех нужных ассетов: ключ, путь, и текстовый
промпт генерации. Минимум покрой: фоны трёх сцен, рамку аватара героя + сам аватар, иконки
золота/кристаллов/SP, рамку и фон карты (по цвету стихии), 3–4 иконки башен (плазма/мороз/
химия/шторм), сегменты шкалы энергии, пиктограмму Реактора, 3–4 спрайта врагов-элементалей,
тайл/декор для карты уровней.

**Рабочий процесс (важен порядок):**
1. Сначала собери **весь каркас на плейсхолдерах** (цветные прямоугольники/контейнеры,
   привязанные к ключам ассетов из манифеста).
2. Затем сгенерируй **первый валидационный батч** из 3–4 ключевых ассетов (фон боя, рамка карты,
   иконка одной башни, аватар) и покажи мне результат — **сверим стиль до массовой генерации.**
3. После подтверждения стиля — догенерируй остальное по манифесту и подмени плейсхолдеры.

Не запускай генерацию всех ассетов разом до сверки стиля (это экономит квоту и время на итерациях).

## 7. Порядок работ

1. Каркас: Vite+TS+Pixi, `ResponsiveLayout`, `SceneManager`, `AssetLoader` с плейсхолдер-фолбэком.
2. Три сцены на плейсхолдерах + навигация между ними.
3. Боевой HUD со всеми элементами из 5.3, включая drag&drop руки и шкалу энергии (моки).
4. `gen_sprite.py` + `assets.manifest.json`; валидационный батч → сверка стиля со мной.
5. Массовая генерация и подмена плейсхолдеров.

После каждого крупного шага — кратко отчитайся, что сделано, и предложи, что дальше.

## 8. Definition of Done

- `npm install && npm run dev` запускается без ошибок.
- Корректно адаптируется на портретном телефоне, планшете и широком экране ПК (без обрезания
  игровой зоны, с тематической рамкой на широких экранах, с учётом safe-area).
- Все три сцены навигируемы; из меню можно дойти до боя и обратно.
- Боевой HUD содержит ВСЕ элементы из 5.3 и реагирует на ввод (моки + логи).
- Спрайт-пайплайн рабочий: `gen_sprite.py` генерирует ассет в едином стиле с прозрачным фоном,
  и хотя бы валидационный батч подключён в игре.
- Краткий `README.md`: как запустить, как сгенерировать ассеты, обоснование версии Pixi и
  ключевых архитектурных решений.
