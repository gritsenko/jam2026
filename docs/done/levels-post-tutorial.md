# ГДД: Три уровня после обучения — прогрессия сложности

> **Статус: MVP реализован.** Per-level волны + тир-множители (`hpScale`/`bountyScale`)
> зашиты в код через [levelCombat.ts](../../src/config/levelCombat.ts)
> (`combatForLevel`), сим множит `maxHp`/`bounty` per-instance при спавне. Три скрипта
> ниже (A/B/C) подключены как уровни **lvl_4 / lvl_5 / lvl_6**; туториальные lvl_1–3 и
> финал lvl_7 дописаны сверх этого дока под доступный на уровне ростер. **Не
> реализованы стретчи** (§6): монстр-саппорт, уникальные модификаторы арены. Сверх
> дока добавлено **направление входа волн** — см. [directional-entry.md](directional-entry.md).
> Текущее поведение — в [../working/current-state.md](../working/current-state.md).

Дев-документ. Три полноценных боевых уровня, идущих **после обучающего кластера**
(уровни, где игрок осваивал установку → мердж → резонанс — см.
[progression-and-tech-tree.md](progression-and-tech-tree.md)). Опирается на баланс
из [enemy-balance.md](../working/enemy-balance.md). Каждый уровень дан готовым скриптом волн в
формате `WaveDef` ([types.ts](../../src/config/types.ts)) — копируется в код почти как есть.

**Сквозная идея прогрессии:** три уровня различаются не «просто больше HP», а
**профилем давления**. Каждый делает доминирующей одну угрозу и проверяет свой навык:

| # | Уровень | Профиль давления | Что проверяет | hpScale |
| --- | --- | --- | --- | --- |
| A | **Ember Hollow** | Брузеры (Magma) | устойчивый DPS, ценность AoE | ×1.15 |
| B | **Glass Dunes** | Саботаж (Disruptor) | uptime башен, Щит, позиция | ×1.10 |
| C | **Coolant Ridge** | Танки + микс + босс | суммарный DPS, защита Core, весь ростер | ×1.30 |

Сложность растёт по двум осям сразу: **профиль усложняется** (одна угроза → микс →
всё+босс) и **тир HP/наград растёт** (×1.15 → ровный, но больше механик → ×1.30).

## 1. Расширение конфига (нужно для уровней)

Сейчас `WAVES` — один общий скрипт на все уровни ([waves.ts](../../src/config/waves.ts)).
Чтобы уровни различались, заводим per-level боевой конфиг с **тир-множителями**
(рычаг §5.3 из [enemy-balance.md](../working/enemy-balance.md) — переиспользуем те же 5 монстров):

```ts
// config/levelCombat.ts (новый)
export interface LevelCombat {
  readonly levelId: string;      // совпадает с LevelNode.id
  readonly waves: WaveDef[];     // скрипт волн этого уровня
  readonly hpScale: number;      // множитель maxHp всех врагов уровня
  readonly bountyScale: number;  // множитель награды (держим ≈ hpScale, чтобы доход успевал)
}
```

Сим при спавне множит `enemy.maxHp × hpScale` и `enemy.bounty × bountyScale`. Базовые
статы остаются в [enemies.ts](../../src/config/enemies.ts); уровень лишь масштабирует тир.
`bountyScale ≈ hpScale` сохраняет гонку дохода (§4 баланс-дока).

> Все бюджеты ниже даны в **базовых HP** (Σ HP без множителя) — это инвариант состава.
> Эффективный бюджет уровня = `базовый × hpScale`.

Базовые HP для расчётов: wisp 35 · volt 55 · magma 90 · disruptor 80 · husk 240.

## 2. Уровень A — Ember Hollow (давление брузерами)

**Фокус:** колонны Magma Brute на фоне фоддера. Учит держать **устойчивый одиночный
DPS** и впервые делает ценным AoE/пробой (Плазма-ударная волна, Рельсотрон). Танк
(Husk) появляется штучно только в финале как намёк на следующий уровень.
**hpScale ×1.15, bountyScale ×1.15. 6 волн.**

```ts
{ levelId: 'lvl_ember_hollow', hpScale: 1.15, bountyScale: 1.15, waves: [
  // W1 — разогрев: бегуны
  { groups: [{ enemyId: 'volt_crawler', count: 6, gap: 0.9 }] },
  // W2 — рой фоддера (учит покрытию)
  { groups: [{ enemyId: 'frost_wisp', count: 10, gap: 0.5 }] },
  // W3 — первые брузеры
  { groups: [
    { enemyId: 'magma_brute', count: 5, gap: 1.3 },
    { enemyId: 'frost_wisp',  count: 6, gap: 0.5 },
  ]},
  // W4 — колонна брузеров + бегуны
  { groups: [
    { enemyId: 'magma_brute',  count: 7, gap: 1.1 },
    { enemyId: 'volt_crawler', count: 6, gap: 0.7 },
  ]},
  // W5 — плотный микс (пик до финала)
  { groups: [
    { enemyId: 'magma_brute',  count: 6, gap: 1.0 },
    { enemyId: 'volt_crawler', count: 8, gap: 0.6 },
    { enemyId: 'frost_wisp',   count: 8, gap: 0.4 },
  ]},
  // W6 — финал: стена брузеров + первые два танка
  { groups: [
    { enemyId: 'magma_brute',  count: 8, gap: 0.9 },
    { enemyId: 'iron_husk',    count: 2, gap: 4.0 },
    { enemyId: 'volt_crawler', count: 8, gap: 0.6 },
  ]},
]}
```

| Волна | Базовый бюджет (Σ HP) | ×1.15 эфф. | рост | Доход (база) |
| --- | --- | --- | --- | --- |
| 1 | 330 | 380 | — | 61 |
| 2 | 350 | 403 | ×1.06 | 75 |
| 3 | 660 | 759 | ×1.89 | 100 |
| 4 | 960 | 1104 | ×1.45 | 124 |
| 5 | 1260 | 1449 | ×1.31 | 167 |
| 6 | 1640 | 1886 | ×1.30 | 177 |

**Star-цель:** ★★★ требует не пропустить ни одного брузера (Core-урон 2 каждый) —
тест на DPS, а не на терпение. Передышка — W2 (рой слабых) после W1.

## 3. Уровень B — Glass Dunes (давление саботажем)

**Фокус:** пачки Disruptor нарастают (2→2→3→4→5) и глушат башни. Сложность здесь —
**не в HP** (поэтому hpScale всего ×1.10), а в **uptime**: без Щита/позиции башни
простаивают. Учит ставить Щит, прятать «тяжёлых» в центр, держать дальнобой.
**hpScale ×1.10, bountyScale ×1.10. 6 волн.**

```ts
{ levelId: 'lvl_glass_dunes', hpScale: 1.10, bountyScale: 1.10, waves: [
  // W1 — разогрев без саботажа
  { groups: [{ enemyId: 'volt_crawler', count: 7, gap: 0.8 }] },
  // W2 — первые саботажники (телеграф механики)
  { groups: [
    { enemyId: 'signal_disruptor', count: 2, gap: 2.5 },
    { enemyId: 'volt_crawler',     count: 6, gap: 0.7 },
  ]},
  // W3 — рой-передышка по HP, но саботаж не уходит
  { groups: [
    { enemyId: 'frost_wisp',       count: 10, gap: 0.4 },
    { enemyId: 'signal_disruptor', count: 2,  gap: 2.0 },
  ]},
  // W4 — пачка саботажников плотнее
  { groups: [
    { enemyId: 'signal_disruptor', count: 3, gap: 1.8 },
    { enemyId: 'volt_crawler',     count: 8, gap: 0.6 },
  ]},
  // W5 — саботаж + брузеры (нельзя простаивать против толстых)
  { groups: [
    { enemyId: 'signal_disruptor', count: 4, gap: 1.5 },
    { enemyId: 'magma_brute',      count: 4, gap: 1.1 },
    { enemyId: 'frost_wisp',       count: 8, gap: 0.4 },
  ]},
  // W6 — финал: рой саботажников + поток + брузеры
  { groups: [
    { enemyId: 'signal_disruptor', count: 5,  gap: 1.2 },
    { enemyId: 'volt_crawler',     count: 10, gap: 0.5 },
    { enemyId: 'magma_brute',      count: 3,  gap: 1.0 },
  ]},
]}
```

| Волна | Базовый бюджет (Σ HP) | ×1.10 эфф. | рост | Disruptor'ов | Доход (база) |
| --- | --- | --- | --- | --- | --- |
| 1 | 385 | 424 | — | 0 | 67 |
| 2 | 490 | 539 | ×1.27 | 2 | 81 |
| 3 | 510 | 561 | ×1.04 | 2 | 99 |
| 4 | 680 | 748 | ×1.33 | 3 | 109 |
| 5 | 960 | 1056 | ×1.41 | 4 | 137 |
| 6 | 1220 | 1342 | ×1.27 | 5 | 165 |

**Ключ:** главный график сложности — **столбец «Disruptor'ов»**, не «бюджет». W3 —
осознанная HP-передышка (рой Wisp), чтобы игрок переставил оборону под саботаж.
**Star-цель:** ★★★ почти невозможна без Щита → уровень «продаёт» Щит как покупку.

## 4. Уровень C — Coolant Ridge (танки + микс + босс)

**Фокус:** бронированные Iron Husk якорят смешанные волны, кульминация — **босс-волна**
из четырёх танков под прикрытием. Учит **суммарному DPS по одиночке**, защите Core и
работе всем ростером + резонансом. Финальный уровень кластера. **hpScale ×1.30,
bountyScale ×1.30. 7 волн** (длиннее предыдущих — climax).

```ts
{ levelId: 'lvl_coolant_ridge', hpScale: 1.30, bountyScale: 1.30, waves: [
  // W1 — брузеры (уже знакомо, но тир выше)
  { groups: [{ enemyId: 'magma_brute', count: 6, gap: 1.0 }] },
  // W2 — поток
  { groups: [
    { enemyId: 'volt_crawler', count: 8, gap: 0.6 },
    { enemyId: 'frost_wisp',   count: 8, gap: 0.4 },
  ]},
  // W3 — первые танки парой
  { groups: [
    { enemyId: 'iron_husk',    count: 2, gap: 3.5 },
    { enemyId: 'volt_crawler', count: 8, gap: 0.6 },
  ]},
  // W4 — танк + саботаж + брузеры (всё сразу)
  { groups: [
    { enemyId: 'signal_disruptor', count: 3, gap: 1.8 },
    { enemyId: 'magma_brute',      count: 6, gap: 0.9 },
    { enemyId: 'iron_husk',        count: 1, gap: 0 },
  ]},
  // W5 — спайк: три танка под прикрытием
  { groups: [
    { enemyId: 'iron_husk',        count: 3,  gap: 3.0 },
    { enemyId: 'volt_crawler',     count: 10, gap: 0.5 },
    { enemyId: 'signal_disruptor', count: 3,  gap: 1.6 },
  ]},
  // W6 — передышка по типам (рой), копим перед боссом
  { groups: [
    { enemyId: 'magma_brute',      count: 8,  gap: 0.8 },
    { enemyId: 'signal_disruptor', count: 4,  gap: 1.5 },
    { enemyId: 'frost_wisp',       count: 12, gap: 0.3 },
  ]},
  // W7 — БОСС-волна: четыре танка + полное прикрытие
  { groups: [
    { enemyId: 'iron_husk',        count: 4,  gap: 2.5 },
    { enemyId: 'magma_brute',      count: 8,  gap: 0.8 },
    { enemyId: 'signal_disruptor', count: 4,  gap: 1.5 },
    { enemyId: 'volt_crawler',     count: 10, gap: 0.5 },
  ]},
]}
```

| Волна | Базовый бюджет (Σ HP) | ×1.30 эфф. | рост | Танков | Доход (база) |
| --- | --- | --- | --- | --- | --- |
| 1 | 540 | 702 | — | 0 | 79 |
| 2 | 720 | 936 | ×1.33 | 0 | 99 |
| 3 | 920 | 1196 | ×1.28 | 2 | 121 |
| 4 | 1020 | 1326 | ×1.11 | 1 | 137 |
| 5 | 1510 | 1963 | ×1.48 | 3 | 195 |
| 6 | 1460 | 1898 | ×0.97 (передышка) | 0 | 199 |
| 7 | 2550 | 3315 | ×1.75 (босс) | 4 | 255 |

**Форма кривой:** ступенчатый рост → спайк (W5) → передышка (W6) → **босс-спайк (W7)**.
Это и есть «напряжение/передышка» из баланс-дока. **Star-цель:** ★★★ = ни один из
4 финальных Husk (Core-урон 3 каждый = 12 из 20) не должен прорваться — жёсткий, но
честный тест на суммарный DPS.

## 5. Прогрессия между уровнями (сводно)

- **A → B → C по профилю:** одиночная угроза (брузеры) → механическая угроза
  (саботаж) → всё сразу + босс. Игрок осваивает по одному ответу за уровень
  (AoE/DPS → Щит/позиция → полный ростер), а не тонет в миксе сразу.
- **A → B → C по тиру:** ×1.15 → ×1.10 → ×1.30. B специально «лёгкий по HP» —
  его сложность в механике, не в статах (иначе саботаж + толстые HP = несправедливо).
  C берёт максимум, потому что игрок к нему уже вооружён всем ростером.
- **Длина:** 6 → 6 → 7 волн. Финальный уровень ощутимо длиннее.
- **Доход растёт с угрозой** через `bountyScale ≈ hpScale` — гонка золота держится
  на каждом тире (§4 баланс-дока).

## 6. Скоуп для прототипа

**MVP (обязательно):**
- `LevelCombat` (§1) + чтение `hpScale`/`bountyScale` в симуляции спавна.
- Три скрипта волн выше как данные; карта мира выбирает нужный по `levelId`.
- Σ HP / доход / Core-бюджет проверены таблицами (уже посчитаны здесь).

**Режем сейчас:**
- Новые типы монстров под эти 3 уровня — **не нужны**, всё на 5 существующих +
  тир-множитель. (Монстр-саппорт из [enemy-balance.md](../working/enemy-balance.md) §6 — отдельный
  стретч, не блокирует эти уровни.)
- Уникальные дороги/арены на уровень — общая арена ок для джема.
- Босс как отдельная сущность — «босс» здесь = волна из 4 танков, а не новый юнит.

**Стретч:**
1. Монстр-саппорт (тематический, §6 баланс-дока) — добавить в C как «дирижёра» пачки.
2. Уникальный модификатор арены на уровень (туман/жара меняет 1 правило).
3. Плейтест-тюнинг множителей по факту прохождения.

## 7. Первый шаг сборки

1. Создать `config/levelCombat.ts` с интерфейсом `LevelCombat` и тремя объектами выше.
2. В [BattleSim](../../src/game/BattleSim.ts) на спавне врага умножать `maxHp` и `bounty`
   на `hpScale`/`bountyScale` активного уровня (прокинуть `LevelCombat` в конструктор
   сима вместо/поверх глобального `WAVES`).
3. Карта мира передаёт `levelId` в бой; бой берёт соответствующий `LevelCombat`.

**Первый конкретный шаг:** завести `config/levelCombat.ts` с уровнем **A (Ember
Hollow)** и прокинуть `hpScale`/`bountyScale` в спавн `BattleSim`. На одном уровне
сразу проверяется и формат данных, и тир-множитель — дальше B и C добавляются как
ещё два объекта.
