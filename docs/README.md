# docs/ — карта документации

Документы разложены по **статусу относительно кода**. Папка = статус. При работе
сначала смотри сюда, чтобы понять, описывает ли док реальность или замысел.

| Папка | Что это | Можно ли верить как описанию игры |
|---|---|---|
| [`working/`](working/) | **Рабочие** — зеркало текущего кода, живут в синхроне с `src/`. | Да, это «как есть сейчас». |
| [`done/`](done/) | **Выполненные** — спеки фич, которые уже зашиты в код. Заморожены, как исторический архив. | Да, но поведение читай в `working/`. |
| [`planned/`](planned/) | **Планируемые** — спроектировано, слейтед к реализации, но в коде **ещё нет**. | Нет — это замысел, не реальность. |
| [`backlog/`](backlog/) | **Беклог** — идеи, ранние/заменённые ревизии, непроритизированное. | Нет — справочно/исторически. |

Не-статусные ассеты лежат в корне `docs/` и **не переезжают** (на них завязаны код
и `tools/`): `cards.json`, `style_ref.png`, `visual_refs/`.

## Правило синхронизации (ВАЖНО)

Доки в `working/` обязаны отражать код. Поэтому:

1. **Меняешь логику боя/экономики/правил** (`src/game/`, `src/config/`) так, что
   меняется наблюдаемое поведение → в том же изменении **обнови соответствующий
   док в `working/`** (как минимум [current-state.md](working/current-state.md)).
2. **Реализовал фичу из `planned/`** → перенеси её спеку в `done/` (заморозка) и
   отрази новое поведение в `working/current-state.md`. Если фича отменена —
   перенеси в `backlog/`.
3. **Завёл новую идею/спеку** → клади в `planned/` (если будем делать) или
   `backlog/` (если «когда-нибудь»). Добавь строку в этот индекс.
4. **При расхождении** код ↔ док в `working/` источник истины — **код**; правь док.

При переносе файла между папками — обнови относительные ссылки (`../`) внутри него
и кросс-ссылки из других доков, а также внешние ссылки из `CLAUDE.md`/кода.
