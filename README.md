# Synergy Grid TD — играбельный каркас

Прототип-оболочка для tower-defense / merge игры на сетке 3×3 («Synergy Grid TD»).
Это **навигируемый каркас** из трёх сцен с полностью свёрстанным боевым HUD на
**замоканных данных** — без боевой логики (синергия, мердж, волны, экономика пока
не реализованы). Архитектура подготовлена под последующий геймплей.

Сделано по [BRIEF.md](BRIEF.md) (= [docs/claude_code_prompt.md](docs/claude_code_prompt.md)),
визуальный референс — [docs/style_ref.png](docs/style_ref.png), данные карт —
[docs/cards.json](docs/cards.json).

## Запуск

```bash
npm install
npm run dev        # http://localhost:5173 (Vite dev-сервер, HMR)
```

Прочие команды:

```bash
npm run build      # tsc --noEmit + продакшн-сборка в dist/
npm run preview    # предпросмотр собранного билда
npm run typecheck  # только проверка типов
```

Требуется Node 18+ (разрабатывалось на Node 25).

## Что уже есть

- **Три сцены и навигация между ними** с fade-переходами:
  - `MainMenuScene` — фон, логотип, витрина платформы, кнопка **START**.
  - `WorldMapScene` — 7 узлов-уровней на вьющейся тропе (пройденные/доступный/
    закрытые), кнопка **BACK**, тап по доступному узлу → бой.
  - `BattleScene` — главный экран со всем боевым HUD (см. ниже).
- **Боевой HUD (§5.3 брифа) на моках:** счётчик волны `WAVE 8/20` и кнопка MAP
  слева сверху; золото, кристаллы и аватар героя в ажурной рамке — справа сверху;
  квадратная арена с дорогой и платформой 3×3 по центру (арена + дорога + враги +
  платформа — один масштабируемый «филд», ничего не разъезжается при ресайзе);
  кликабельные слоты; сегментированная **шкала энергии** (зелёный→жёлтый→красный +
  золотой Overdrive); рука из 3 карт (иконка/нагрузка/рамка по цвету стихии); зона
  **Реактора** у правого края — появляется только во время перетаскивания карты.
  Очки синергии (SP) пользователю не показываются.
- **Drag & drop** карты из руки: при перетаскивании подсвечиваются валидные
  drop-зоны (пустые слоты + Реактор), при отпускании — визуальный фидбек и лог
  (мок), карта возвращается в руку.
- **Адаптивность**: телефон (узкий портрет), планшет, широкий экран ПК — без
  обрезания игровой зоны и без чёрных полос (подробнее ниже).
- **Спрайт-пайплайн** с подменой плейсхолдера на готовый PNG по ключу ассета.

## Структура

```
index.html              # full-screen canvas, safe-area, no-zoom
vite.config.ts
tsconfig.json           # strict + noUncheckedIndexedAccess
src/
  main.ts               # точка входа: роуты сцен → Game.boot()
  theme.ts              # дизайн-токены: палитра, цвета стихий, шрифты, DESIGN-разрешение
  core/
    Game.ts             # bootstrap: Application, ассеты, layout, ресайз, кадровый цикл
    SceneManager.ts     # активная сцена + fade-переходы
    ResponsiveLayout.ts # адаптивный лейаут (см. ниже)
    AssetLoader.ts      # загрузка спрайтов + плейсхолдер-фолбэк по ключу
    scene.ts            # базовый класс Scene + SceneServices
    tween.ts            # минимальный твин на тикере Pixi (без зависимостей)
  ui/                   # переиспользуемые компоненты (Button, BattleCard, EnergyGauge,
                        #   PlatformGrid, SlotView, ReactorZone, HeroAvatar, ResourceChip,
                        #   WaveBadge, EnemySprite, WorldMapNode, SceneBackground, helpers)
  scenes/               # MainMenuScene, WorldMapScene, BattleScene
  config/               # ДАННЫЕ отдельно от логики: cards, enemies, levels,
                        #   battleState (мок), assetManifest, types
assets/sprites/         # готовые PNG (подключаются по ключу)
tools/                  # gen_sprite.py + assets.manifest.json (см. tools/README.md)
```

## Ключевые архитектурные решения

- **PixiJS v8.19** (запинено в `package.json`). Выбор: рендер на WebGL2/WebGPU,
  лучшая производительность на мобильных, актуальная сцена-граф/ассет-архитектура
  v8 (`Application.init`, `Assets`, новый Graphics API). Используем WebGL
  (`preference: 'webgl'`) для широкой совместимости.
- **Vite 7 + TypeScript 5.9, strict.** Взяты стабильные прошлые мажоры (не самые
  свежие 8/6), чтобы у jam-прототипа не было сюрпризов с тулчейном; Pixi —
  единственная запиненная на «последнюю» зависимость, как просил бриф. Менеджер
  пакетов — **npm**.
- **Без твин-библиотеки.** `core/tween.ts` — крошечный твин на `app.ticker`
  (fade-переходы, всплывания карт, пульсации). «Чистый Pixi + TS», ноль внешних
  зависимостей рантайма, кроме самого Pixi.
- **`core/ResponsiveLayout`** (свой, не `@pixi/layout`). Весь UI верстается в
  портретном дизайн-пространстве `1080×…`:
  - **Портретные экраны** (телефон) — вписывание по ширине, высота «резиновая»
    (в пределах clamp): HUD-бар липнет к реальному верху, рука — к низу, а не
    тупо масштабируется целиком.
  - **Широкие/ландшафтные** (планшет, ПК) — портретная игровая «рамка»
    центрируется, свободные поля заполняются тематическим фоном/декором (брасс-
    рамка вокруг зоны), **без чёрных полос**.
  - Учитываются `env(safe-area-inset-*)` (чёлки/скругления) и `devicePixelRatio`
    (через `autoDensity`, с ограничением сверху для перфа). Корректная обработка
    `resize`/смены ориентации без перезагрузки сцены.
- **`SceneManager`** держит одну активную сцену и делает fade-переход через
  оверлей в экранных пикселях (всегда покрывает весь canvas).
- **`AssetLoader` с плейсхолдер-фолбэком.** Игра ссылается только на **ключи**
  ассетов. Реальные PNG из `assets/sprites/` подхватываются через `import.meta.glob`
  Vite; если файла нет — строится тематический плейсхолдер из манифеста. Подмена
  плейсхолдера на сгенерированный спрайт = просто появление файла (с перезапуском
  dev-сервера), без правок кода. Есть мягкие фолбэки (напр. фон меню/карты временно
  берёт `bg_level`).
- **Данные отдельно от отрисовки** — всё в `src/config/` как типизированные
  структуры; рендер-компоненты их только читают.

## Управление

- **Меню:** START → карта уровней.
- **Карта:** тап по доступному (подсвеченному) узлу → бой; BACK → меню.
- **Бой:** перетаскивайте карту из руки на пустой слот или в Реактор (подсветка
  зон + лог в консоли); тап по слоту — лог; кнопка MAP → назад на карту.

## Графика и спрайт-пайплайн

Весь арт — сгенерированные PNG (никаких эмодзи/CSS-фигур), см.
[tools/README.md](tools/README.md) и [CLAUDE.md](CLAUDE.md). Источник —
Gemini «Nano Banana» через `tools/gen_sprite.py`; единый стиль держится преамбулой
в `tools/sprite_style.py` + авто-референсом `docs/style_ref.png`.

**Статус ассетов.** Валидационный батч уже принят и подключён: фон арены
(`bg_level`), платформа (`base_platform`, как витрина в меню), турель плазмы
(`plasma_shutter`), иконки ресурсов (`icon_gold/crystal/synergy/energy`) и UI-чром
(`ui_button`, `ui_panel`, `ui_card_frame`, `ui_button_overdrive`). Остальные ключи
(5 карт, враги, аватар героя + рамка, узлы карты, отдельные фоны меню/карты) пока
показываются плейсхолдерами и ждут массовой генерации — список и промпты в
[tools/assets.manifest.json](tools/assets.manifest.json) и `src/config/assetManifest.ts`.

Сгенерировать недостающий ассет (имя файла = ключ):

```powershell
tools\.venv\Scripts\python.exe tools\gen_sprite.py "frost pulse turret, icy blue emitter coils" assets\sprites\frost_pulse.png --category card_icon --size 512
```

## Чего здесь намеренно нет

Боевой логики: атак, синергии стихий, мерджа карт, расчёта нагрузки/перегрузки,
волн и экономики. HUD реагирует на ввод, но цифры статичны (моки из
`config/battleState.ts`). Это следующий слой поверх готового каркаса.
